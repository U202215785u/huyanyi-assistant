# AI 反问 重设计 — 设计文档

日期：2026-05-07

## 概述

将当前自由聊天式 AI 反问工具改造为结构化的卡片选模式 + 分步骤引导流程。核心方法论不变：通过反问驱动头脑风暴。"反问"不再是独立模式，而是两种模式共享的底层方法。

## 两种模式

| 模式 | 图标 | 描述 |
|------|------|------|
| 想法发散 | 💡 | 从用户起点出发，通过反问挖掘维度、边界、可能，最终展开具体方向 |
| 提示词优化 | ✨ | 通过反问了解需求，生成结构化可复用的提示词，支持迭代打磨 |

## 架构

```
桌面宠物（原地动画）→ 点击呼出窗口 → 欢迎页（二选一卡片）→ 分步引导
```

### 欢迎页

- 顶部标题 "AI 反问"，副标题说明反问方法论
- 两个大卡片各占一行：想法发散 / 提示词优化
- 每张卡片：图标 + 标题 + 描述 + 能力标签
- hover 边框高亮，点击进入对应模式
- 底部保留设置和导出按钮

### 分步引导流程

**💡 想法发散（3 步）**

```
步骤 1 — 输入起点：用户输入想法起点
步骤 2 — 反问挖掘（循环）：AI 给选项，用户点选，逐轮深入，不限轮次
步骤 3 — 发散展开：基于全部对话，展开具体方向、变体、跨界组合
```

**✨ 提示词优化（3 步）**

```
步骤 1 — 描述需求：用户描述想让 AI 完成的任务
步骤 2 — 反问澄清（循环）：AI 给选项，了解目标/受众/格式/风格/约束
步骤 3 — 生成 & 打磨：输出完整提示词，支持微调迭代
```

### 反问交互模式（步骤 2 核心）

每轮反问：
- AI 给出 3~4 个选项 + "其他（自己写）"
- 用户点选项 → 直接进入下一轮追问
- 用户点"其他" → 展开输入框自由打字
- 底部始终有"挖够了，进入下一步"按钮
- 不限轮次，用户控制何时结束

### 进度条 & 导航

- 窗口顶部固定进度条：`← 返回 | ● 起点 → ◎ 反问 → ○ 发散`
- `← 返回` 回到欢迎页，保留已选模式的步骤进度
- 已完成步骤可点击跳回
- 底部按钮随步骤变化：
  - 步骤 1：`[ 下一步 → ]`
  - 步骤 2：`[ 继续挖掘 ]` / `[ 挖够了 → ]`
  - 步骤 3：`[ 重新开始 ]` / `[ 回到首页 ]`

## 桌面宠物

- 保留 Neko 精灵动画
- **不再追逐鼠标**，固定在原地循环原地动作：坐 → 洗脸 → 打盹 → 睡觉 → 坐...
- 点击：惊讶动作 → 400ms 后呼出对话窗口
- 仍然可拖动

## 数据流

- 会话级对话历史保存在内存，窗口关闭时写入 `~/.ai-fanwen/history.json`
- 每个模式独立保存步骤状态和对话记录
- 模式切换需确认清空当前进度
- API：DeepSeek Chat，每次请求携带 system prompt + 完整历史

## 待删除

- 标题栏文字模式标签（反问/想法/提示词）
- 打字的占位文输入框 → 改为步骤 1 的专用输入区
- 欢迎页曾经的 "把想法告诉我 + 我会通过层层反问" 文案 → 改为欢迎卡片

## 滚动到完成

- 欢迎页两张卡片 + 点击呼出
- 3 步流程切换（步骤视图容器）
- 反问交互（选项按钮 + "其他"输入框 + "挖够了"）
- 进度条 + 回退导航
- 两种模式的 system prompt 适配选项式反问输出
- 步骤 3 发散展开 / 生成提示词
